01 — Overview

Project Summary

This document is the single source of truth for two junior front-end developers building a professional website for an aluminum installations company (κουφώματα, τζαμαρίες, πόρτες ασφαλείας, στέγαστρα, κιγκλιδώματα). The prior planning agent produced comprehensive client-discovery questions and a technology comparison. This blueprint synthesises both into a concrete, actionable specification.

Core goal: Generate inbound leads (phone calls + appointment form submissions) from local homeowners, architects, and construction contractors. Everything — architecture, copy, CTA placement, SEO — serves this one goal.
🎯

Primary Audience

Local homeowners renovating, construction contractors, and architects seeking technical aluminum solutions.

📍

Geographic Scope

Initially local/regional. Architecture must support multi-city expansion without restructuring.

⚡

Success Metric

Measurable increase in appointment-form submissions and tracked phone-call conversions within 90 days of launch.

🔒

Phase 2 Ready

Architecture is designed to accommodate B2B partner portal (professionals/contractors) in a future phase without rebuilding.

03 — Technology

Recommended Tech Stack

Based on the planning agent's JAMstack recommendation and calibrated to two junior developers, here is the final, opinionated stack with rationale for every decision.

⚛️

Next.js 14 (App Router)

React framework with SSG + ISR. Pages are pre-rendered at build time — lightning fast, SEO-perfect. App Router enables React Server Components for zero client JS on static pages.

Primary SSG / ISR
🧠

Sanity.io (Headless CMS)

Structured content with real-time preview. Client can update portfolio, edit text, upload photos from a clean dashboard — no code required. Free tier covers this project.

Primary Free Tier OK
🚀

Vercel

Zero-config deployment for Next.js. Automatic HTTPS, global CDN, preview deployments on every Git branch push. Hobby plan is free and sufficient for this site's traffic.

Primary Free Tier OK
🔐

Clerk (Auth — Phase 2)

Handles user accounts, role management, Google/Facebook sign-in. Pre-built React components mean junior devs can add auth in hours, not days. Defer to Phase 2 unless B2B portal is in scope now.

Phase 2 Optional
📧

Resend + React Email

Appointment form submissions trigger transactional emails to the business owner. Resend's free tier supports 3,000 emails/month — more than enough.

Required Free Tier OK
🎨

Tailwind CSS v4

Utility-first CSS. Combined with a custom design token layer, ensures consistent spacing, color, and typography across all pages. Much faster for two developers to stay in sync.

Required
🖼️

Cloudinary (Images)

Sanity's built-in CDN handles most images, but Cloudinary provides advanced transforms: auto WebP/AVIF conversion, lazy-load blur placeholders, and responsive srcset generation.

Recommended Free Tier OK
📊

Plausible Analytics

Privacy-first analytics (GDPR-compliant, no cookie banner needed). Tracks form submissions, page views, referrers. Lightweight script doesn't hurt Core Web Vitals.

Recommended
Why not WordPress? For two junior developers, WordPress seems approachable but becomes a liability. Plugin conflicts, PHP security patches, slow image-heavy pages, and poor Core Web Vitals scores all hurt SEO and the client relationship long-term. The JAMstack approach has a steeper initial learning curve but produces a dramatically faster, safer, and more maintainable site.

// app/lib/sanity.ts — GROQ query example for projects
import { createClient } from 'next-sanity'

export const client = createClient({
  projectId: process.env.NEXT_PUBLIC_SANITY_PROJECT_ID,
  dataset:   'production',
  apiVersion: '2024-01-01',
  useCdn: true,
})

// Fetch all projects with filtering support
export async function getProjects(category?: string) {
  const filter = category
    ? `&& category == "${category}"`
    : ''

  return client.fetch(`
    *[_type == "project" ${filter}] | order(date desc) {
      _id, title, slug, category,
      mainImage { asset->{ url, metadata { lqip, dimensions } } },
      location, summary, date
    }
  `)
}

07 — Auth & Roles

Authentication Strategy

Phase 1 recommendation: Ship without user accounts. The primary goal is lead generation, not a membership portal. Auth adds 2–3 weeks of development time and complexity that diverts focus from the core deliverable. Plan Phase 2 auth architecture now — but don't build it yet.

Phase 2 Role Architecture (Clerk)

When the client is ready to open a B2B portal for contractors and architects, implement the following role model using Clerk + Sanity metadata:

👤

Guest (Unauthenticated)

Full access to all public pages: portfolio, services, blog, contact. Can submit appointment form. No login required.

🏠

Registered User

Saves favourite projects. Downloads technical PDFs (brochures). Pre-filled contact forms. Access to member-exclusive offers.

Self-register
🔧

Professional / Partner

Wholesale price lists. CAD drawings. Priority appointment booking. Direct project inquiry channel. Requires admin approval.

Admin approved
⚙️

Admin

Full CMS access via Sanity Studio. Approves professional accounts. Views analytics dashboard. Manages email campaigns per role.

Internal only

  '/dashboard(.*)',
  '/technical-drawings(.*)',

08 — SEO Strategy

Search Engine Optimisation

For a local aluminium business, SEO is the primary long-term traffic channel. The architecture is designed to maximise local search visibility from day one.

<1sTarget LCP (Largest Contentful Paint)
100Target Lighthouse Performance Score
AVIFImage format (60% smaller than JPEG)
3Structured data schemas implemented

Local SEO (Most Important)

Google Business Profile must be claimed and linked. Each service page targets keywords like "κουφώματα αλουμινίου + [πόλη]". Schema: LocalBusiness with address, phone, hours.

Structured Data

LocalBusiness on every page. FAQPage on /faq and service pages (rich snippets). BreadcrumbList on all inner pages. Implemented via Next.js JSON-LD component.

Image SEO

Every project image gets descriptive alt text from Sanity: "Συρόμενα αλουμινίου Alumil στη Γλυφάδα". Google Images is a significant traffic source for renovation products.

Content Strategy

10 blog articles targeting long-tail keywords: "τι είναι θερμοδιακοπή", "αλλαγή κουφωμάτων κόστος", "πόσο αντέχει αλουμίνιο". These pages attract high-intent visitors ready to buy.

09 — Performance

Performance Optimisation

Performance is not optional — it directly impacts Google ranking and user conversion rates. A 1-second delay in page load reduces conversions by ~7%.

✓
next/image for every image. Automatic WebP/AVIF conversion, responsive srcset, blur placeholder from Sanity's LQIP (Low Quality Image Placeholder) metadata. Never use a raw <img> tag.
✓
Static Generation (SSG) for all public pages. Zero server processing time per request. Pages served as pre-built HTML from Vercel's edge CDN.
✓
Font subsetting. Use next/font/google — it downloads only Greek character subset, removes render-blocking, and self-hosts the font file at build time.
✓
Lazy-load below-the-fold content. Portfolio gallery images load only when scrolled into view. Use loading="lazy" on gallery components.
✓
ISR (Incremental Static Regeneration). Portfolio and blog pages revalidate every 60 seconds after a Sanity publish webhook fires. No rebuild needed for the whole site.
!
Third-party scripts. Load Google Maps embed with loading="lazy" via iframe facade. Delay Plausible analytics script until after first user interaction.
!
Contact form. Use a Server Action (Next.js 14) instead of a client-side fetch — no extra JavaScript bundle shipped to the browser for form handling.

11 — Sprint Plan

6-Week Development Plan
