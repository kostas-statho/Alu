Κίνδυνοι & Αντιμετώπιση
⚠️

Καθυστέρηση Παράδοσης Assets από Πελάτη

Ο πελάτης δεν στέλνει φωτογραφίες ή κείμενα εγκαίρως. Αυτό είναι ο Νο1 κίνδυνος σε κάθε web project στην Ελλάδα.

Ξεκαθαρίστε εγγράφως στη σύμβαση: "Καθυστέρηση assets > 3 ημέρες = ανάλογη παράταση χρονοδιαγράμματος." Δώστε deadline με ημερομηνία, όχι "σύντομα".
🔄

Αλλαγές Scope ("Και ένα ακόμα...")

Ο πελάτης ζητά extra λειτουργίες που δεν συμφωνήθηκαν (π.χ. "μπορείτε να βάλετε και e-shop;").

Σύμβαση με explicit scope. Κάθε αλλαγή εκτός scope = νέα προσφορά. Πείτε το φιλικά αλλά ξεκάθαρα από την αρχή.
📸

Ανεπαρκείς ή Κακής Ποιότητας Φωτογραφίες

Ο πελάτης έχει μόνο φωτογραφίες από κινητό (low res, κακός φωτισμός). Το portfolio θα φαίνεται ερασιτεχνικό.

Ενημερώστε εγκαίρως: "Για επαγγελματικό αποτέλεσμα χρειαζόμαστε min 1200px." Προτείνετε επαγγελματία φωτογράφο (~€200-400).
🔧

Τεχνικά Εμπόδια σε Junior Επίπεδο

Η ομάδα είναι junior. Κάποια features (ISR webhooks, Server Actions, Sanity schemas) μπορεί να πάρουν περισσότερο χρόνο από το αναμενόμενο.

Buffer 20% στο κάθε sprint. Χρησιμοποιήστε τα Sanity + Next.js docs και τα guides του project. Μην "εφεύρετε τον τροχό" — υπάρχουν έτοιμα examples.
📱

Mobile Performance Προβλήματα

Εικόνες χωρίς next/image, βαριά fonts, ή πολλά third-party scripts μπορούν να "σκοτώσουν" τη βαθμολογία Lighthouse σε mobile.

Κανόνας: ΠΑΝΤΑ next/image. ΠΑΝΤΑ next/font. Third-party scripts (Maps) με lazy load. Lighthouse audit από την S4, όχι μόνο στο τέλος.
💬

Ασαφής Επικοινωνία μεταξύ Developers

Δύο devs χωρίς συντονισμό μπορούν να δουλεύουν σε conflicting components ή να έχουν inconsistent code style.

Daily 10-λεπτο standup (async ή live). Git branching strategy: main / develop / feature-*. ESLint + Prettier από S1 για ενιαίο style.
